    react-router-dom源码解析(八) -NavLink

    前天学习了Link组件源码, 今天再来看一下与之相似的NavLink

    前言

    经过上一篇文章的战火洗礼, 可以了解到, Link作为沟通react-router和react之间的桥梁, 经历click -> url变化 -> history(onListen) -> Switch -> Route render的过程, 据此, 有关Link的面试题便可迎刃而解

    那么, 有了Link的前车之鉴, 去学习NavLink便能少几分疑惑.

    细说

    正式开始分析之前, 先来回想一下我们平时是如何使用NavLink的?

    传入对应的activeStyle或者activeClass, 点击之后, 如果对应的组件成功渲染, 则该NavLink的样式也会改变.

    带着平时使用的思路去分析源码, 会又事半功倍的效果.
